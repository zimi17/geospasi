# ADR-002: Strategi Publikasi Data Spasial dengan Biaya Nol

**Tanggal:** 2026-06-07
**Status:** Disetujui
**Pemrakarsa:** Arsitek & Product Manager

---

## Judul: Publikasi data spasial via GitHub RAW CDN — Bukan streaming langsung dari Supabase

**Konteks:**
- Supabase free tier memiliki batas bandwidth 2 GB/bulan
- Jika peta web dibuka banyak orang (publik, Pemkab, media), bandwidth bisa habis dalam hitungan hari
- Tim tidak memiliki anggaran untuk langganan berbayar
- Data agregat yang ditampilkan ke publik tidak perlu real-time (cukup update harian/mingguan)

**Keputusan:**
Arsitektur publikasi data berlapis:

```
[Admin/ETL] → Supabase (database internal, akses terbatas)
                    ↓ (GitHub Actions, scheduled atau trigger)
              GeoJSON agregat → commit ke repo
                    ↓
              GitHub RAW CDN (unlimited bandwidth, gratis)
                    ↓
              [Publik] → Leaflet.js di GitHub Pages
```

- Data agregat (ringkasan per desa, tanpa data individu) di-export sebagai GeoJSON
- GeoJSON di-commit ke repo oleh GitHub Actions (manual trigger atau schedule)
- Frontend Leaflet membaca GeoJSON dari `raw.githubusercontent.com` (CDN, tanpa batas bandwidth)
- Supabase hanya diakses oleh admin/peneliti — bandwidth 2 GB tidak akan habis

**Dampak:**
- (+) Publikasi data tidak terbatas bandwidth — GitHub RAW tidak punya kuota
- (+) Tidak perlu membayar apapun — seluruh stack gratis
- (+) Data agregat tetap version-controlled di Git
- (-) Data publik tidak real-time (delay 1 commit)
- (-) Butuh GitHub Actions workflow untuk export otomatis

**Catatan:**
Jika di masa depan dibutuhkan data real-time, opsi gratis tetap ada:
- Cloudflare Workers (100k request/hari gratis) sebagai API proxy
- Oracle Cloud Always Free (VM 24GB RAM) hosting API sendiri

Keputusan ini tidak perlu diubah — tambahkan layer real-time di atasnya jika diperlukan.
